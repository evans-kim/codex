# 여보, 오늘 뭐 먹지? 🍽️

오늘의 기분과 몸 상태에 어울리는 저녁 메뉴를 이미지와 함께 추천하는 브라우저 전용 웹앱입니다.

**배포 앱:** <https://evans-kim.github.io/codex/>

## 주요 기능

- 기분, 체력, 당기는 맛, 식사 방식의 4단계 선택
- 30개 메뉴를 대상으로 한 설명 가능한 점수 기반 추천
- 대표 메뉴 1개와 대안 2개, 새 메뉴 다시 추천
- 프로젝트에 포함된 전용 음식 SVG 일러스트
- 좋아요·별로예요, 최근 식사 중복 감점, 선택 기록
- Web Share API와 클립보드 복사 대체
- JSON 취향 데이터 백업·복원·초기화
- 홈 화면 설치와 오프라인 실행을 위한 PWA
- 모바일·데스크톱 반응형 UI와 키보드 접근성

## 개인정보 설계

기분, 선호도, 식사 기록은 현재 브라우저의 `localStorage`에만 저장됩니다. 서버, 외부 API, 광고, 방문 분석 도구를 사용하지 않습니다. 메뉴 이미지와 코드 역시 저장소의 정적 파일만 불러옵니다.

브라우저 데이터나 사이트 저장 공간을 삭제하면 기록도 함께 삭제됩니다. 필요한 경우 설정에서 JSON 파일로 내보낸 뒤 복원할 수 있습니다.

## 실행

빌드 단계나 패키지 설치가 필요하지 않습니다. 정적 서버만 실행하세요.

```bash
python3 -m http.server 4173
```

브라우저에서 `http://localhost:4173`으로 접속합니다. `file://`로 직접 열면 ES Modules와 Service Worker가 브라우저 보안 정책에 의해 제한될 수 있습니다.

## 테스트

Node.js 20 이상에서 외부 패키지 없이 실행됩니다.

```bash
npm test
```

테스트 범위는 추천 규칙, 최근 메뉴 감점, 저장 데이터 정제, 메뉴·아이콘 파일 경로, Service Worker 캐시, 외부 런타임 URL 부재를 포함합니다.

## 추천 로직

메뉴의 최종 점수는 대략 다음 요소로 계산합니다.

```text
기분 적합도 + 체력 적합도 + 음식 느낌 + 식사 방식 + 추가 조건
+ 좋아요 학습 - 최근 메뉴 중복 - 충돌 조건 + 소량의 다양화
```

상위 후보에서 음식 카테고리가 겹치지 않도록 세 메뉴를 고르기 때문에 같은 종류만 반복되는 현상도 줄였습니다. 의료·심리 진단은 수행하지 않습니다.

## 배포

`main` 브랜치가 소스 브랜치이며, 검증된 동일 커밋을 `gh-pages` 브랜치에도 게시합니다. GitHub Pages는 `gh-pages` 브랜치의 루트 정적 파일을 서비스합니다. 배포 전에는 `npm test`와 브라우저 스모크 테스트를 실행합니다.
